
## Организационные моменты хакатона Tupolev IT Challenge

## [Понимаю что это выглядит как спагетти код, поэтому пишу документацию... к документации, ладно.]


1. В данном документе описаны основные моменты по организации хакатона **Tupolev IT Challenge 2026 далее** Хакатон
2. В директории Helpers описаны помощники по организации и вопросы которые возникают к ним и далее по документации будут встречаться такие же обозначения, сохраняющие свой смысл.
	1. [ ] - незакрытый вопрос
	2. [x] закрытый вопрос
3. В директории **Images** располагаются изображения, прилагаемые в контексте, обязательно нужно указывать ссылки на них.
4. В директории **Normative** располагаются основные моменты по организации
	1. Достижения **ITupolev** за все время существования
	2. Концепция Хакатона
	3. Даты проведения, дедлайны сдачи и тд
	4. Для кого создается данный Хакатон
	5. Количественные достижения **ITupolev** за все время существования (Количество **вступивших/принявших** участие человек)
	6. Необходимые ресурсы для организации Хакатона
	7. Ожидаемый результат проведенного мероприятия
	8. **Основное положение порядка проведения Хакатона**
5. В директории **Partners** располагаются партнеры, помогающие в организации Хакатона
	1. Эксперты - также коллегия жюри, делегаты от сторонних компаний и тд. 
	2. Финансовая поддержка (она заключается не только в денежном эквиваленте, но и в виде мерча КАИ/Заинтересованных лиц)
	3. Мерч КАИ - описано где и с кем нужно договариваться чтобы его выделили
	4. Другие партнеры (сторонние лица), привлеченные на Хакатон вне КНИТУ-КАИ (инвесторы, эксперты и тд)
6. В директории **Presentation** описаны все пункты связанные с презентацией
7. В директории **ProblemsAndSolutions**
	1. Проблемы обнаруженные в ходе проведения Хакатонов прошлых лет 
	2. Проблемы обнаруженные в ходе подготовки к проведению Хакатонов прошлых лет
8. В директории **SMI** описаны различные пиар-компании
	1. Матрица - медиа сообщество студенческого совета ИКТЗИ
	2. Медиа цифровой кафедры КНИТУ-КАИ 
	3. Медиа большого совета КНИТУ-КАИ
9. В директории **Steck** описаны допустимые технологии
	1. Основные положения допустимости используемой технологии
	2. Стек для трека №1
	3. Стек для трека №2
	4. Стек для трека №3
10. В директории **Style** описана общая стилистика Хакатона (обсуждается с дизайнерами, копирайтерами)
11. В директории **ThemeOfTracks** описана тематика треков 
12. В файле TupolevITChallenge2026 описаны основные моменты для подготовки и организации Хакатона
