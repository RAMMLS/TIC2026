## Тематика треков

Тематика треков должна пересекаться с тематикой хакатона.
 
Если хакатон не будет ни к чему приурочен, то треки будут выставляться по заказу заинтересованных лиц [[OtherPartners]] 

будет представлено 3 трека различной сложности 

### Трек 1: Оптимизация производственных цепочек (Data Science/Backend)

**Задача:** Разработать бэкенд-сервис для моделирования и оптимизации логистики поставок компонентов на сборочный конвейер. Требуется минимизировать время простоя оборудования за счет эффективного планирования поставок, учитывая вероятности задержек.

- **Примерная сложность:** Средняя. Требует навыков в алгоритмах оптимизации и бэкенд-разработке.
- **Технологический стек:** Python (Pandas, Scikit-learn или собственные алгоритмы), SQL/NoSQL БД, Docker, FastAPI/Django.
- **Результат:** Рабочий API-сервис, который принимает данные о поставках и выдает оптимальный график.

### Трек 2: Цифровой двойник узла самолета (Frontend/VR/GameDev)

**Задача:** Создать интерактивную 3D-модель (цифровой двойник) конкретного авиационного узла (например, шасси или части двигателя), которая позволяет в режиме реального времени визуализировать данные о его состоянии (температура, давление) или проводить интерактивное обучение по сборке/разборке.

- **Примерная сложность:** Средняя. Нужны навыки 3D-визуализации и работы с движками.
- **Технологический стек:** Unity/Unreal Engine (C#/C++), Three.js или Babylon.js для веб-версии, WebSocket для получения данных.
- **Результат:** Интерактивное приложение (десктоп, веб или VR-прототип), демонстрирующее модель в действии.

### Трек 3: Система мониторинга полетных данных БПЛА (Embedded/IoT/Fullstack)

**Задача:** Создать прототип системы для сбора и визуализации телеметрических данных с условного БПЛА. Задача включает в себя эмуляцию устройства, передачу данных по защищенному каналу и разработку дашборда для их отображения в реальном времени.

- **Примерная сложность:** Средняя. Требует понимания передачи данных и работы с "железом" (пусть и эмулированным).
- **Технологический стек:** C++/Python для эмуляции/встроенного ПО, MQTT/HTTP для связи, любая БД, React/Vue/Angular для дашборда.
- **Результат:** Работающая система, где данные с "устройства" отображаются на интерактивном веб-дашборде.
